# NSZPC 功能用法文件

這份文件提供「前台使用方式」與「後台管理流程」，可直接給營運或內容編輯人員參考。

## 1. 使用角色

- 前台訪客：瀏覽推薦配單、分類總覽、近期出機與技術文章。
- 後台管理員：登入後編輯網站內容、配單、分類、出機紀錄、文章與庫存資料。

## 2. 啟動方式

1. 在專案根目錄啟動資料庫

```bash
docker compose up -d
```

2. 啟動 backend

```bash
cd backend
npm install
cp .env.example .env
npm run db:init
npm run dev
```

3. 啟動 frontend

```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```

4. 常用網址
- 前台：`http://localhost:5173`
- 後端 API：`http://localhost:3000`
- Swagger：`http://localhost:3000/api-docs`
- pgAdmin：`http://localhost:5050`

## 3. 前台功能用法

### 3.1 首頁 `/`

首頁目前包含以下區塊：

- 推薦配單：點卡片可進入配單詳細頁。
- 近期出機：顯示近期案例，支援點標籤前往標籤總覽。
- 出貨流程：顯示標準服務流程。
- 客戶回饋：顯示 testimonial。
- 聯絡我們：顯示聯絡管道與地圖。

### 3.2 分類總覽 `/categories`

- 可用標籤與關鍵字篩選分類。
- 每張分類卡可進入分類專屬頁 `/categories/item/:id`。

### 3.3 分類詳情 `/categories/item/:id`

- 顯示分類的需求說明、重點、FAQ。
- 會顯示「你可能有興趣的主機」推薦區塊，導向配單詳情。

### 3.4 推薦配單詳情 `/builds/item/:id`

- 顯示電腦介紹、需求說明、核心配備與價格。
- 支援 YouTube 嵌入（若後台有填連結）。

### 3.5 出機標籤總覽 `/orders/tags`

- 點擊標籤可篩選符合該標籤的主機。
- 支援多標籤同時篩選（AND 條件）。
- 點擊案例可進入出機詳情頁 `/orders/item/:id`。

### 3.6 出機詳情 `/orders/item/:id`

- 顯示客戶需求、完整配備、出貨資訊與標籤。
- 支援 YouTube 嵌入（若後台有填連結）。

### 3.7 技術文章 `/blog` 與 `/blog/:slug`

- 列表頁可用標籤 + 關鍵字篩選文章。
- 文章詳情頁包含段落內容、標籤與影片區塊。
- 文章頁已套用 SEO metadata 與 JSON-LD。

## 4. 後台功能用法

### 4.1 登入

- 入口：`/admin/login`
- 預設帳號密碼使用 `backend/.env` 的 `ADMIN_USERNAME` / `ADMIN_PASSWORD`。

### 4.2 後台主控台 `/admin`

主控台包含多個管理分頁：

- 網站內容管理
  - 編輯首頁文案、流程、客戶回饋、聯絡資訊。
  - 管理「分類快速標籤」與「近期出機統一標籤庫」。
- 推薦配單管理
  - 新增/編輯/刪除推薦配單。
  - 維護需求描述、規格、圖片與 YouTube 連結。
- 分類總覽管理
  - 新增/編輯/刪除分類資料。
  - 維護標籤、重點、FAQ、分類詳情圖片與說明。
- 近期出機管理
  - 新增/編輯/刪除出機紀錄。
  - 可套用統一標籤庫，並產出估價單。
- 同行拿貨紀錄
  - 管理每日多筆拿貨項目，追蹤結清狀態與統計。
- 公司進貨紀錄
  - 管理公司進貨明細與供應商統計。
- 庫存管理
  - 管理品項分類、數量、末端價格與成本。

### 4.3 技術文章管理 `/admin/blog`

- 新增/編輯/刪除文章。
- 必填欄位：`slug`、標題、摘要、封面、發布日期、段落內容。
- 可填：標籤、作者、閱讀分鐘數、YouTube 嵌入連結。
- 發布後前台 `/blog` 與 `/blog/:slug` 會同步讀取。

## 5. 標籤管理規則

- 出機標籤建議先在「網站內容管理 > 近期出機統一標籤庫」維護。
- 編輯出機資料時，優先從統一標籤庫勾選，避免同義詞或大小寫混亂。
- 前台 `/orders/tags` 會合併「統一標籤庫 + 既有出機資料標籤」作為篩選來源。

## 6. 文章 SEO 使用建議

新增或編輯文章時建議：

- `slug` 使用英文小寫與連字號，例如 `rtx-5070-build-guide`。
- 標題含主要關鍵字與用途情境。
- 摘要控制在 60-120 字內，描述使用者可獲得的資訊。
- 標籤維持 3-8 個，避免過度堆疊。
- 優先提供可公開存取的封面圖與 YouTube 影片。

系統目前會自動輸出：

- `title`、`description`、`canonical`
- Open Graph / Twitter meta
- JSON-LD（Blog、BlogPosting、Breadcrumb）

## 7. 常見操作 SOP

### 7.1 新增一筆近期出機案例

1. 前往 `/admin` > `網站內容管理`，確認統一標籤庫已存在需要的標籤。
2. 前往 `近期出機管理`，點 `新增近期出機`。
3. 填入日期、需求描述、配備、售價、地區、標籤與影片連結。
4. 儲存後到前台 `/orders/tags` 檢查標籤篩選是否正確。

### 7.2 發布一篇技術文章

1. 前往 `/admin/blog`。
2. 點 `新增文章`，填入 slug、標題、摘要、封面、段落內容。
3. 儲存後到前台 `/blog` 與對應 `/blog/:slug` 檢查顯示與 SEO。

### 7.3 新增一個分類與對應推薦配單

1. 在 `/admin` 的 `分類總覽管理` 新增分類資料。
2. 在 `推薦配單管理` 建立對應配單並補齊需求與規格。
3. 前往前台 `/categories/item/:id` 驗證「你可能有興趣的主機」是否合理。
